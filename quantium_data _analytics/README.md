# 🛒 Retail Analytics – Quantium Virtual Experience

## 📌 Overview
This project is part of the Quantium Data Analytics Virtual Experience on Forage. The objective was to analyze retail transaction data to generate business insights and evaluate the impact of store layout changes.

---

## 🎯 Objectives
- Understand customer purchasing behavior
- Identify key customer segments
- Evaluate the effectiveness of trial store layouts
- Provide data-driven business recommendations

---

## 📊 Dataset
The dataset contains transaction-level data including:
- Store number
- Customer ID
- Product details
- Sales value
- Customer segmentation (Life stage & Affluence)

---

## ⚙️ Tools & Technologies
- Python
- Pandas
- NumPy
- Matplotlib

---

## 📈 Key Analysis

### 🔹 Task 1: Customer & Category Insights
- Identified key customer segments driving sales
- Analyzed purchasing patterns based on life stage and affluence
- Found that Budget and Mainstream customers contribute most to sales

---

### 🔹 Task 2: Trial vs Control Store Analysis
- Selected control stores using correlation analysis
- Compared trial stores (77, 86, 88) with control stores
- Metrics used:
  - Total Sales
  - Number of Customers
  - Transactions per Customer

---

## 📉 Key Findings
- Trial stores did not show a significant uplift in sales compared to control stores
- Observed changes were consistent with normal business variation
- Correlation alone is not sufficient; store scale must also be considered

---

## 💡 Recommendations
- Do not roll out the new store layout at this stage
- Conduct further testing with better-matched control stores
- Extend the trial period for deeper insights

---

The project includes:
